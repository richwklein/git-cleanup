# Security Policy

## Reporting a vulnerability

If you have discovered a security vulnerability in this project, please report it through GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Fill in the advisory form with as much detail as possible — reproduction steps, affected versions, and potential impact.

You will receive an acknowledgement within a reasonable timeframe. Please do not file a public issue or PR for security problems.

## Supported versions

Only the latest released version receives security updates. Older versions are not patched.
